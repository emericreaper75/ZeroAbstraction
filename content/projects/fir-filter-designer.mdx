## Overview

The FIR Filter Designer is a browser-based tool for designing finite impulse response digital filters. It runs entirely client-side — no server required — with the computationally intensive Parks-McClellan (Remez exchange) algorithm compiled from Rust to WebAssembly for near-native performance.

**Why this exists:** Existing FIR design tools require either a MATLAB/Python installation (barrier to entry) or use suboptimal window methods (Kaiser, Hamming) that produce filters with $10$–$15\%$ more coefficients than necessary for the same specification. By putting the equiripple-optimal Parks-McClellan algorithm in the browser with real-time D3.js frequency response rendering at 60 fps, engineers can iterate on filter specifications interactively.

> **Note.** The Parks-McClellan algorithm guarantees the minimum filter order for a given set of passband ripple, stopband attenuation, and transition bandwidth specifications. For a typical lowpass design with $0.1\,\text{dB}$ passband ripple and $60\,\text{dB}$ stopband attenuation, Parks-McClellan requires order $N = 63$ vs. $N = 71$ for the Kaiser window method — an $11\%$ reduction in computational cost per filtered sample.

## Parks-McClellan Background

### The Minimax Problem

Given a desired frequency response $D(\omega)$, a weighting function $W(\omega)$, and a set of constrained frequency bands $F$ (passbands and stopbands, excluding transition bands), find the Type I linear-phase FIR filter $H(\omega)$ of order $N$ that solves:

$$
\min_{h[n]} \; \max_{\omega \in F} \left| W(\omega) \left[ D(\omega) - H(\omega) \right] \right|.
$$

For a symmetric FIR filter ($h[n] = h[N-n]$), the frequency response has the form:

$$
H(\omega) = e^{-j\omega N/2} \sum_{k=0}^{M} a[k] \cos(k\omega), \quad M = \lfloor N/2 \rfloor,
$$

reducing the problem to finding $M+1$ real coefficients $\{a[k]\}$.

### The Chebyshev Alternation Theorem

The optimal solution exists, is unique, and satisfies the **alternation theorem**: the weighted error achieves its maximum magnitude at $\geq M+2$ distinct frequencies $\omega_1 < \omega_2 < \cdots$ with alternating sign:

$$
W(\omega_i)\left[D(\omega_i) - H(\omega_i)\right] = (-1)^i \delta, \quad i = 1, \ldots, M+2,
$$

where $\delta$ is the equiripple error level. This characterisation is both necessary and sufficient for optimality, meaning any filter satisfying the alternation condition is guaranteed to be the unique minimax-optimal solution.

### Filter Order Comparison

For a lowpass filter with passband edge $f_p = 0.2 f_s$, stopband edge $f_s = 0.25 f_s$, passband ripple $\delta_p = 0.01$ ($0.087\,\text{dB}$), and stopband attenuation $\delta_s = 0.001$ ($60\,\text{dB}$):

## WebAssembly Core

The Remez exchange algorithm is implemented in Rust and compiled to WebAssembly via `wasm-pack`. The core function:

```rust
use wasm_bindgen::prelude::*;
use std::f64::consts::PI;

#[wasm_bindgen]
pub fn design_fir(
    order: usize,
    bands: &[f64],      // [f1_start, f1_end, f2_start, f2_end, ...]
    desired: &[f64],     // [d1, d2, ...] one per band
    weights: &[f64],     // [w1, w2, ...] one per band
    max_iter: usize,
) -> Vec<f64> {
    let n_coeffs = order + 1;
    let m = order / 2;  // Type I assumption (even order)
    let n_extremals = m + 2;
    
    // Initialise extremal frequencies uniformly across bands
    let mut extremals = init_extremals(bands, n_extremals);
    let mut coeffs = vec![0.0; n_coeffs];
    
    for _iter in 0..max_iter {
        // Step 1: Build and solve the interpolation system
        // The Chebyshev alternation condition gives a linear system:
        // sum_k a[k] cos(k * omega_i) + (-1)^i * delta / W(omega_i) = D(omega_i)
        let (a, delta) = solve_alternation_system(
            &extremals, m, bands, desired, weights
        );
        
        // Step 2: Evaluate weighted error on dense frequency grid
        let n_grid = 2048;
        let (grid, errors) = evaluate_error_on_grid(
            &a, n_grid, bands, desired, weights
        );
        
        // Step 3: Find new extremal set via exchange
        let new_extremals = find_extremals(&grid, &errors, n_extremals);
        
        // Step 4: Check convergence
        let max_change = extremals.iter()
            .zip(new_extremals.iter())
            .map(|(a, b)| (a - b).abs())
            .fold(0.0_f64, f64::max);
        
        extremals = new_extremals;
        
        if max_change < 1e-12 {
            break;
        }
    }
    
    // Recover filter coefficients via IDFT of the cosine polynomial
    let (a, _) = solve_alternation_system(
        &extremals, m, bands, desired, weights
    );
    recover_coefficients(&a, n_coeffs)
}
```

The convergence criterion $\max |\Delta\omega_i| < 10^{-12}$ ensures coefficient accuracy to machine precision. Typical convergence occurs in 15–25 iterations for filter orders up to $N = 512$.

## Usage: Designing a 60 dB Stopband Lowpass

1. **Set filter type** to "Lowpass" in the specification panel
2. **Set passband edge** to $f_p = 0.2$ (normalised to Nyquist)
3. **Set stopband edge** to $f_s = 0.25$
4. **Set passband ripple** to $0.1\,\text{dB}$ ($\delta_p = 0.0115$)
5. **Set stopband attenuation** to $60\,\text{dB}$ ($\delta_s = 10^{-3}$)
6. **Click "Design"** — the WASM engine computes coefficients in $<1\,\text{ms}$
7. **Review** the magnitude response (dB), phase response, and group delay plots in the D3.js canvas
8. **Export** coefficients as JSON, C header array, or MATLAB `.mat` file

The tool automatically estimates the minimum filter order using the Bellanger approximation:

$$
N \approx \frac{-2}{3} \log_{10}\left(10 \delta_p \delta_s\right) \cdot \frac{f_s}{f_s - f_p}
$$

and rounds up to the nearest even integer for Type I symmetry.
